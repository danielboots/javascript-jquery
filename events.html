animation with JQuery

JQuery effects are how the changes to html appear visibly to the user

Effects such as hide, show, slide, and fade can be implemented over a specified time resulting in animation

How do you use it? .show(1000); will show an item in 1000 milliseconds

transcript

Animation with jQuery Up to now, we have seen simple animations in CSS. jQuery supports even more powerful animation features out-of-the-box. The ability to fade elements in and out of visibility or to slide elements can be done with a simple line of code. All the effects can be implemented over a specified time, resulting in animation. The duration of the effect may specify the milliseconds, or use one of three preset timings: fast, medium, or slow. So let's look at some of these effects. The show() method can show a hidden element, such as a div or an image element. Here's a simple example. Here, we're passing a speed to the show() method. It tells jQuery how fast to show the element. So we have three predefined: slow, medium, and fast; or a number of milliseconds, in this case 1000. An element can be hidden using the hide() method, or, with CSS, using display:none. The hide() method makes an element vanish. It's the opposite of show(). And here are some of the speeds we can choose from: slow, medium, fast, or, again, a preset millisecond time. Let's have a look at some of these examples on jQuery. If we have a look at show(), as soon as we press this button, we will see the paragraph. And with hide(), as soon as we press the button, we will hide the paragraph. The jQuery toggle() function is very useful. It toggles the visibility of an element. Therefore, if the element is hidden, it will be shown, and, vice-versa, if it is shown, it will be hidden. It works just like an on/off button. If you think about using the toggle() function, it does not make much sense to use the function on an element itself. When the toggled element becomes hidden, you cannot click on it again to make it visible. So what we usually do is click on an element, such as a button, which toggles a different element. The slideDown() function slides down a hidden HTML element, so it becomes visible. As with the show() and hide() functions, you can also set the speed as a parameter to the function. The opposite to the previous function is the slideUp() function, which hides a visible element by sliding it up. The slideToggle() function shows an element if it's hidden, or hides it if it's visible. The fadeIn() function fades in the hidden element. Like with the show and hide functions, you can also set the speed as a parameter to the function. The fadeOut() function fades out a visible element. The fadeToggle() function shows an element if it's hidden, or hides it if it's visible. The fadeTo() function enables you to fade an HTML element partially in or out, thus making it transparent. You pass two parameters: the speed, which is in milliseconds; and the opacity, which is between zero and one, which tells you how much you wish to fade the element. 0 is fully transparent (hidden) and 1 is fully opaque (shown). Method chaining A wonderful aspect of jQuery is the fact that most of the methods return an object that you can then use to call another method. This allows you to do command chaining where you can perform multiple methods on the same set of elements. This can be very useful and can save a lot of time. So let's look at how this might be done. If we want to remove a class from an element and add a different class, we would normally do the following. What we can do here is chain these two together. Chaining also works with events. So let's have a look at an example with events. Here, we have the slideUp() followed by the slideDown() chained together. An element's attributes can contain useful information, so it's important to be able to access and manipulate them. The attr() function acts both as a getter and a setter. As with the CSS function, we can query with one parameter, here we're looking for an href, and update the attribute for the second parameter, in this case, to example.com

Method Chaining With jQuery, means you can chain together many actions/methods. Chaining gives us the ability to run multiple jQuery methods on a single element while using only a single statement.

$('p').removeClass("make-border").addClass("make-red");

This can also be written like shown below, and is still considered valid format, but for testing stick to using the single line version. 

$('p').removeClass("make-border")
        .addClass("make-red");

traversing the dom tree

the syntax for getting the parent 

 $('#thebutton').parent();

You can obtain the direct parent element of a selected element by calling a parent() method. This method traverses up a single, or parent level, of the DOM tree.

Traversing down the DOM tree with the children function The children() function returns all direct child elements of a given element. This method only traverses down a single level. So the syntax here would be $, and then, say, div, for example, in parenthesis in quotes, .children. If you wanted to return a specific child element, for example, paragraphs, then you would put "p" in the children method in quotes. Now go ahead and try the challenges.
